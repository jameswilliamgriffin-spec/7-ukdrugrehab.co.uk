# Project Brief — ukdrugrehab.co.uk (Site 7)

## Network Context
This site is part of a seven-site microsite network 
supporting The Wellbourne Clinic. It is the regional/local 
SEO counterpart to drugsrehabuk.com (site 6), mirroring 
the relationship between ukalcoholrehab.com and 
alcoholrehabuk.com.

Hub site: https://thewellbourneclinic.co.uk/

## This Site's Purpose
Local and regional SEO for drug rehab and detox. 
City and region-specific pages targeting "drug rehab [location]" 
search intent across the West Midlands and surrounding areas.

## Tone
- Clear, direct, locally relevant
- Practical — people searching these terms want to know 
  what's near them and how to start
- Still calm and clinical in register, consistent with 
  the rest of the network
- Avoid generic "near me" filler — each location page 
  should feel genuinely specific where possible

## Stack
- Next.js (App Router)
- Tailwind CSS
- Framer Motion for scroll animations and interactions
- Match the project structure and conventions used in sites 1-6

## Brand Guidelines
- Primary brand colour: #F19885
- Wellbourne Clinic logo in header and footer
- Typography: Plus Jakarta Sans (display) + DM Sans (body)
- Same colour palette as established for the network

## Page Structure

### Core Pages
1. Hero — local/regional drug rehab focus
2. What Is Drug Rehab? (short overview, links to site 6 pillar page)
3. Why Choose Local Treatment (proximity, family involvement, aftercare continuity)
4. Our Service Area (map or list of covered regions)
5. FAQ Section (schema markup)
6. Final CTA

### Location Pages (one per priority location)
Each follows a consistent template:
1. Localised hero — "Drug Rehab in [Location]"
2. Brief local context (why local treatment matters)
3. What treatment involves
4. Getting started / next steps
5. Internal links to relevant site 6 substance guides
6. FAQ specific to that location where relevant
7. CTA

## Important
- This site should NOT duplicate the substance-specific 
  cluster content from site 6 — it links out to that content 
  rather than repeating it
- Location pages must each have genuinely distinct content, 
  not the same template with the city name swapped — 
  this is the single biggest risk for this site type
- Do not build every location at once — start with the 
  5 priority locations, validate the approach, then expand
